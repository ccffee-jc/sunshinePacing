# 技术设计: GUI 实时突发图表

## 技术方案
### 核心技术
- Fyne 自绘组件（canvas/raster）绘制折线图
- GUI 定时采样 `Metrics()` 并维护环形缓冲

### 实现要点
- 采样周期固定 500ms。
- 维护三条序列（video/control/audio）各自的环形缓冲（例如 120 点≈60秒）。
- video 使用 `Metrics().VideoQueueLen` 作为曲线值。
- control/audio 使用 `StatsSnapshot` 的增量字节（Δbytes/500ms）作为突发强度。
- 用三种颜色绘制折线并在图表区域标注图例。

## 架构设计
保持 core 逻辑不变，仅在 GUI 侧采样与绘制。

## 安全与性能
- **性能:** 500ms 刷新 + 环形缓冲限制点数；绘图放在 GUI 线程安全更新。
- **安全:** 不涉及外部输入与敏感数据存储。

## 测试与部署
- **测试:** 本地运行 GUI，观察曲线随负载变化；构建产物验证。
- **部署:** 无额外步骤，随 GUI 构建产物发布。
