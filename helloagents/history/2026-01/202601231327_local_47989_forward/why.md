# 变更提案: 本机47989端口族直转到内网Sunshine

## 需求背景
当前默认 internal_offset 为 1000，会导致外部端口基于 47989+1000。需求希望外部基准端口保持 47989，并将该端口族转发到 192.168.2.110 上对应的 47989 端口族，同时提供可用的配置与编译产物。

## 变更内容
1. 允许 internal_offset 取 0，使外部端口与 Sunshine 原始端口一致。
2. 提供面向 192.168.2.110 的配置样例，基准端口 47989。
3. Linux 环境编译 CLI 产物，输出到与配置同目录。

## 影响范围
- **模块:** config / core / cli
- **文件:** internal/config/config.go, proxy.yml, scripts/build.sh(不改动)
- **API:** 无
- **数据:** 无

## 核心场景

### 需求: 端口族直通映射
**模块:** config/core
将本机基准端口 47989 的端口族直接映射到 192.168.2.110 的 47989 端口族。

#### 场景: internal_offset = 0
外部端口与 Sunshine 原始端口一致（如 http=47989、web=47990、video=47998 等）。
- 预期结果1：本机监听的外部端口与 Sunshine 端口一致
- 预期结果2：转发目标指向 192.168.2.110 对应端口族

### 需求: Linux 编译产物与配置同目录
**模块:** cli
将 Linux CLI 二进制输出到配置文件同目录，便于直接运行。

#### 场景: 本地编译
在项目根目录生成 sunshine-proxy。
- 预期结果1：编译成功输出到项目根目录
- 预期结果2：使用 proxy.yml 可直接启动

## 风险评估
- **风险:** internal_offset=0 放大了端口冲突概率（本机已有服务占用 47989 族）。
- **缓解:** 提醒确认端口占用；启动失败时提示调整端口或先停用冲突服务。
